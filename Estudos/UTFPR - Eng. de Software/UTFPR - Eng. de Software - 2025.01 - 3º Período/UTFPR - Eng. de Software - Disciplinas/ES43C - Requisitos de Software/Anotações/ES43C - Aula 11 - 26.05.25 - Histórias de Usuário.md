## 🌳Histórias de Usuário

Os Modelos de Processo de Software são os responsáveis pela criação das Histórias de usuário. Criado inicialmente no XP e aprimorado pelo Scrum. 

### 🌿XP

O XP é um metodologia para pequenas equipes que irão desenvolver softwares com constantes mudanças. Defendem a revisão de código, integração rápida, teste automatizados, feedback do cliente e design simples. Tem como Valores a comunicação, simplicidade, o feedback, a coragem e o respeito.  

A documentação em um projeto que usa XP é focada, principalmente, no próprio código acompanhada por seus testes automatizados, Além de códigos e testes pouco material extra é produzido. Apenas cartões de história do usuário e alguns diagramas e gráficos de acompanhamento. 

### 🌿Scrum

Tem como artefatos, as histórias de usuário, casos de uso, itens de negócio ou funcionais, itens não funcionais, arquiteturas, infraestrutura, etc. Tem como parte da documentação a Lista ordenada de requisitos desejados para o produto em forma de histórias do usuário.

### 🌿03. Rastreabilidade de requisitos

É a possibilidade de rastrear os requisitos ao longo de todo o ciclo. É o processo. Busca identificar e documentar o elo que envolve um determinado requisito para que seja possível rastrear sua origem. 

- Compreender a evolução do artefato
- Resolver conflitos entre requisitos 
- Verificar requisitos nos processos de testes 
- Corrigir defeitos através da identificação do componente de origem do erro 
- **Analisar o impacto das mudanças nos requisitos** 
- Gerenciar riscos e reuso de componentes
- Compreender os aspectos do projeto 
- Estimar variações em cronogramas e em custos do projeto

#### 🌱Capacidade

- Forward → rastreamento do requisitos inicial (gerador) até seu refinamento
- Backward → rastreamento do requisito final (refinado) até seu gerador

#### 🌱Classificação

- Pré-especificação → os artefatos são a fonte ou a origem de um requisito 
- Entre os requisitos → mapeamento de dependência entre os requisitos. 
- Pós-especificação → requisitos oriundos de atividades subsequentes de desenvolvimento e/ou teste.

### 🌿Matriz de Rastreabilidade

Busca facilitar a visualização dos relacionamento entre requisitos e outros artefatos ou objetos. Coloca-se os objetos sendo rastreados nos eixos de uma tabela e marca-se os pontos de intersecção.

Se o número de requisitos a ser gerenciado for grande as tabelas de requisitos serão problemáticas. Poderá ser usada uma forma simplificada de rastreamento onde ao lado da descrição dos requisitos, são mantidas uma ou mais listas de identificadores dos requisitos relacionados

### 🌿04. Ferramentas

O gerenciamento de requisitos envolve a coleta, armazenamento e manutenção de grande quantidade de informação. Existe um grande número de ferramentas disponíveis que foram projetadas para suportar o gerenciamento de requisitos, porém a maioria são pagas. Outras ferramentas, tais como, sistemas de gerenciamento de configuração podem ser adaptadas para a engenharia de requisitos

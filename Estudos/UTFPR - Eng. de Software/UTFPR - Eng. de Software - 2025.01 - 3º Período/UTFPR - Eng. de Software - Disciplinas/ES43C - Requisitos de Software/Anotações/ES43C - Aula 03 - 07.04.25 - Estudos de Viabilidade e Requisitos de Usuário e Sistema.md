## 🌳Estudos de Viabilidade

Um estudo de viabilidade decide se vale ou não a pena gastar tempo e esforço com o sistema proposto. Busca saber se o sistema contribui para os objetivos da organização, pode ser implementado usando a tecnologia atual, o cronograma e o orçamento e se o sistema pode ser integrado

- Viabilidade de Cronograma → *Dado o atual conhecimento, os prazos podem ser cumpridos?*
- Viabilidade Econômico → *É uma avaliação de custo eficiência de um projeto ou solução*
- Viabilidade Técnica → *É a avaliação prática de uma solução técnica específica*
- Viabilidade Operacional → *É uma medida do grau da solução para a organização*

 Após isso, o relatório pode, propor mudanças, sugerir outros requisitos ou cancelar o projeto
 
### 🌿Requisitos de Usuário

São declarações ==em linguagem natural== somada a diagramas, dos serviços que se espera do sistema. Esses requisitos podem variar de declarações amplas das características necessárias do sistema até descrições precisas e detalhadas

### 🌿Requisitos de Sistema

São declarações mais detalhadas das funções, dos serviços, e das restrições do sistema. O Documento de requisitos deve ==definir exatamente== o que deve ser implementado. Pode fazer parte do contrato entre o adquirente do sistema e os desenvolvedores de software

### 🌿Stakeholders 

Os stakeholders incluem qualquer um que seja afetado de alguma maneira pelo sistema e, portanto, tenha um interesse legítimo nele. Podem variar de usuários finais de um sistema a gerentes e stakeholders externos, como autoridades reguladoras, que certificam a aceitabilidade do sistema.

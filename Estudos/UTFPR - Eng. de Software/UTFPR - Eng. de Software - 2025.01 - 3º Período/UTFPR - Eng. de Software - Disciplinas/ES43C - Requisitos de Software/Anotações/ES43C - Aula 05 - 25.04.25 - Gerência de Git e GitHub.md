## 🌳Gerenciamento de Configuração 

Em projetos em equipe é essencial que vários devs trabalhem ao mesmo tempo em sistema de software. Como as mudanças em projeto de desenvolvimento podem ocorrem em qualquer instante, as atividades de Gerência de Configuração tornam-se importante para:
- Identificar, controlar e acompanhar as mudanças;
- Registrar a evolução do sistema;
- Assegurar que a alteração seja implementada corretamente; e
- Relatar as alterações.

O Gerenciamento de configuração está relacionado com as políticas, processos e ferramentas para gerenciamento de mudanças dos sistemas de software. Quando aplicado ao desenvolvimento de software é baseado em um conjunto de atividades. 
- Controle de Mudanças;
- Controle de Versões;
- Construção do Sistema; e
- Gerenciamento de Releases. 

### 🌿Controle de Mudanças

 Está relacionado com a análise de custos e benefícios das mudanças propostas, a aprovação e o acompanhamento dos componentes alterados. 

 Problema identificado > Análise do problema > Análise de mudanças e custos > Implementação e testes > Artefatos revisados.
 
### 🌿Controle de Versão

Envolve manter o acompanhamento de versões de componentes e assegurar que as mudanças nos componentes, realizadas por diferentes devs, não interfiram umas nas outras.

### 🌿Construção do Sistemas

Processo de criação de um sistema completo, executável por meio da construção e ligação dos componentes do sistema, bibliotecas externas, arquivos de configuração, etc.

### 🌿Gerenciamento de Releases

Um release de sistema é uma versão de um sistema distribuída aos clientes. Envolve a preparação de software para o release externo e manter o acompanhamento das versões de sistema que foram liberadas para uso do cliente. As atualizações também são chamadas de releases. Como parte de uma metodologia ágil, geralmente os releases sçao liberados ao final de cada ciclo de trabalho.

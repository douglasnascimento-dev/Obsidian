### 🌳Priorização de Requisitos

 É essencial de requisitos é essencial para garantir que os recursos do projeto sejam aplicados de forma estratégica. 

 - **Essencial** → A sua falta torna o software inaceitável para as partes. | ALTO
 - **Desejáveis** → Agrega um valor substancial, mas sua falta não torna o software inaceitável. | INTERMEDIÁRIO
 - **Opcionais** → O valor agregado é pequeno, o produto seria pleno mesmo sem ele. | BAIXO

### 🌿Técnica Moscow

- **M** → Deve ter - obrigatório ou imprescindível
- **S** → Deveria ter - é importante, mas não imprescindível
- **C** → Poderia ter - não é essencial, mas seria bom
- **W** → Não terá - não agrega valor ao negócio

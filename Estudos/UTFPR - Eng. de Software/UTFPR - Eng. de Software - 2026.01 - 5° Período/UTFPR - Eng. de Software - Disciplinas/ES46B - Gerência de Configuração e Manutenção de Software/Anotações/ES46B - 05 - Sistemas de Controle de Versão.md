## 🌿 BUILD

É o processo de compilação completa do Software, onde o Código Fonte é transformado em um Artefato Executável. Representa uma versão ainda incompleta do software em desenvolvimento, mas com certa estabilidade. Costuma apresentar limitações conhecidas. 

## 🌿RELEASE

É um novo empacotamento do Software, é o Produto Final. Existem Releases Internos (para teste, para as equipes) e Externos (para implantação do Cliente). Mas nem todo Build vira um Release.

## 🌿REPOSITÓRIO

É a área no Servidor responsável em controlar o versionamento dos Arquivos. Para desenvolvedor, o repositório é mapeado em um sistema de diretórios.  Como exemplo comum, um repositório hospedado no GitHub.

## 🌿WORKING DIRECTORY

É a Pasta Local onde os arquivos do Projeto estão sendo editados localmente no computador do desenvolvedor. Reflete a estrutura do servidor de versões e é nesta estrutura que o desenvolvedor modificada arquivos. 

## 🌿CLONE

É a Operação realizada pelo desenvolvedor, e cria um espelho do repositório em seu PC. 

## 🌿UPDATE

É a Operação que atualiza os Arquivos do Desenvolvedor.

## 🌿ADD

É a Operação que adiciona os Arquivos ao Sistemas de Controle de Versão, pois mesmo que um Arquivo esteja em uma pasta versionada, se este arquivo não estiver adicionado, ele não irá compor o versionamento, mas esse comando não envia o Arquivo ao Servidor.

## 🌿COMMIT

É a Operação de Atualização de Arquivos e consequentemente gera uma nova versão.

## 🌿LOCK

É a Operação que o Desenvolvedor bloqueia o arquivo e evita que demais desenvolvedores o modifiquem. O Git não possui a função Lock.

## 🌿BRANCH

É uma Linha Alternativa de criação do projeto

## 🌿MERGE

É a Operação que une arquivos que foram modificados por dois desenvolvedores
